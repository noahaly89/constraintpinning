# Constraint Registry

The registry is the authoritative source for constraint metadata and immutable versions.

Minimum capabilities:

- unique identifier enforcement;
- version history and integrity checks;
- approval and lifecycle state;
- ownership and review dates;
- machine-readable retrieval;
- access control;
- dependency and conflict references;
- complete audit trail.

Runtime systems should cache only with explicit freshness and invalidation rules.
