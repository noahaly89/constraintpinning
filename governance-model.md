# Governance Model

A practical governance model includes five responsibilities:

| Role | Responsibility |
|---|---|
| Constraint owner | Defines intent and remains accountable for the requirement. |
| Approver | Authorizes activation or material change. |
| Registry custodian | Maintains integrity, metadata, versioning, and access. |
| Implementer | Integrates validation and enforcement into the system. |
| Exception authority | Decides whether a time-bound deviation is permissible. |

Exceptions should be explicit objects with scope, expiration, justification, compensating controls, and approval.
